## 可能导致注册失败的原因

### 1. Supabase 邮箱验证设置（最常见）

**问题：** Supabase 默认要求邮箱验证，用户注册后必须点击验证邮件才能登录。

**解决方案：**

#### 方案A：关闭邮箱验证（推荐用于测试）
1. 登录 Supabase 控制台：https://supabase.com/dashboard
2. 选择你的项目：`yospqbbeykizesujlhiq`
3. 进入 `Authentication` → `Providers` → `Email`
4. 找到 `Confirm email` 选项
5. **取消勾选** "Confirm email"
6. 点击 Save 保存

这样用户注册后可以直接登录，无需验证邮箱。

#### 方案B：配置邮件服务（推荐用于生产环境）
1. 在 Supabase 控制台进入 `Project Settings` → `Auth`
2. 配置 SMTP 设置或使用 Supabase 默认邮件服务
3. 确保邮件模板正确配置
4. 测试邮件发送功能

### 2. 注册限制设置

**问题：** Supabase 可能禁用了公开注册功能。

**解决方案：**
1. 进入 Supabase 控制台
2. `Authentication` → `Policies`
3. 确保 "Enable email signups" 已启用
4. 检查是否有 IP 限制或其他安全策略

### 3. 邮箱已被注册

**问题：** 用户使用的邮箱已经注册过。

**现在的处理：** 代码会显示 "该邮箱已被注册，请直接登录"

### 4. 密码强度不足

**问题：** 密码少于6个字符。

**现在的处理：** 代码会在提交前验证并提示 "密码至少需要6个字符"

### 5. 邮箱格式错误

**问题：** 邮箱格式不正确。

**现在的处理：** 代码会验证邮箱格式并提示 "请输入有效的邮箱地址"

### 6. 频率限制

**问题：** 短时间内多次注册尝试。

**现在的处理：** 代码会显示 "发送邮件过于频繁，请稍后再试"

## 测试步骤

### 1. 测试注册功能
```
1. 打开网站
2. 在右上角输入邮箱和密码（密码至少6位）
3. 点击"注册"按钮
4. 观察提示信息：
   - 如果显示"注册成功"弹窗 → 注册功能正常
   - 如果显示错误信息 → 根据错误信息排查
```

### 2. 检查浏览器控制台
```
1. 按 F12 打开开发者工具
2. 切换到 Console 标签
3. 尝试注册
4. 查看是否有错误信息
5. 截图发送给开发者
```

### 3. 检查 Supabase 后台
```
1. 登录 Supabase 控制台
2. 进入 Authentication → Users
3. 查看是否有新注册的用户
4. 检查用户的验证状态（Confirmed 列）
```

## 常见错误信息对照表

| 英文错误信息 | 中文提示 | 解决方案 |
|------------|---------|---------|
| User already registered | 该邮箱已被注册，请直接登录 | 使用其他邮箱或直接登录 |
| Invalid email | 邮箱格式无效 | 检查邮箱格式 |
| Password should be at least 6 characters | 密码长度不足 | 使用至少6位密码 |
| Signups not allowed | 当前不允许注册，请联系管理员 | 检查 Supabase 设置 |
| Email rate limit exceeded | 发送邮件过于频繁，请稍后再试 | 等待几分钟后重试 |

## 推荐配置（用于测试）

为了方便测试，建议临时使用以下配置：

1. **关闭邮箱验证**（见上文方案A）
2. **启用自动确认**
   - 在 Supabase 控制台
   - `Authentication` → `Settings`
   - 启用 "Enable email confirmations"

3. **测试账号**
   - 使用真实邮箱进行测试
   - 或使用临时邮箱服务（如 temp-mail.org）

## 用户使用指南

### 注册流程
1. 在右上角输入邮箱和密码（至少6位）
2. 点击"注册"按钮
3. 如果需要邮箱验证：
   - 查收邮箱（包括垃圾箱）
   - 点击验证链接
   - 返回网站登录
4. 如果无需验证：
   - 直接登录使用

### 邮箱验证邮件未收到？
- 检查垃圾邮件/垃圾箱文件夹
- 等待几分钟后刷新邮箱
- 确认邮箱地址是否正确
- 检查邮箱是否已满
- 联系管理员重新发送
