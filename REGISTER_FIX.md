## 🎯 使用方法

### 方式一：使用邮箱注册（推荐）
1. 打开 `register.html`
2. 在"邮箱地址"字段输入真实邮箱（例如：`user@example.com`）
3. 输入密码（至少6个字符）
4. 确认密码
5. 点击"立即注册"

### 方式二：使用用户名注册（测试）
1. 打开 `register.html`
2. 在"邮箱地址"字段输入用户名（例如：`testuser`）
3. 系统会自动转换为 `testuser@jiuzhi.test`
4. 输入密码并确认
5. 点击"立即注册"

## ⚠️ 重要提示

### Supabase 邮箱验证设置

**如果注册后无法登录，可能是邮箱验证设置问题：**

#### 方案A：关闭邮箱验证（推荐用于测试）
1. 登录 [Supabase 控制台](https://supabase.com/dashboard)
2. 选择项目：`yospqbbeykizesujlhiq`
3. 进入 `Authentication` → `Providers` → `Email`
4. 找到 `Confirm email` 选项
5. **取消勾选** "Confirm email"
6. 点击 Save 保存

这样注册后可以直接登录，无需验证邮箱。

#### 方案B：配置邮件服务（生产环境）
1. 在 Supabase 控制台进入 `Project Settings` → `Auth`
2. 配置 SMTP 设置
3. 确保邮件模板正确配置

### 测试邮箱处理

如果使用测试邮箱（`@jiuzhi.test`）：
- 不会收到验证邮件
- 需要在 Supabase 控制台的 `Authentication` → `Users` 中查看用户
- 如果启用了邮箱验证，需要手动在控制台验证用户

## 🔍 验证注册是否成功

### 方法一：查看浏览器控制台
1. 打开浏览器开发者工具（F12）
2. 查看 Console 标签
3. 注册成功会显示：`注册成功: {user: {...}}`

### 方法二：查看 Supabase 控制台
1. 登录 Supabase 控制台
2. 进入 `Authentication` → `Users`
3. 查看是否有新注册的用户

### 方法三：尝试登录
1. 注册成功后
2. 使用注册的邮箱和密码尝试登录
3. 如果登录成功，说明注册正常

## 🐛 常见问题排查

### Q1: 点击注册按钮没有反应？
**检查：**
1. 打开浏览器控制台（F12），查看是否有 JavaScript 错误
2. 确认 `auth.js` 和 Supabase 库已正确加载
3. 检查网络连接

### Q2: 显示"该邮箱已被注册"？
**解决：**
- 该邮箱已经注册过，请直接登录
- 或使用其他邮箱注册

### Q3: 显示"当前不允许注册"？
**解决：**
1. 检查 Supabase 控制台设置
2. 进入 `Authentication` → `Policies`
3. 确保 "Enable email signups" 已启用

### Q4: 注册成功但无法登录？
**可能原因：**
1. Supabase 启用了邮箱验证，需要先验证邮箱
2. 密码输入错误
3. 用户被禁用

**解决：**
1. 检查 Supabase 控制台的邮箱验证设置
2. 如果使用测试邮箱，在控制台手动验证用户
3. 或关闭邮箱验证功能

## 📋 注册流程

```
用户填写表单
    ↓
前端验证（用户名/邮箱格式、密码长度、密码确认）
    ↓
转换为邮箱格式（如果是用户名）
    ↓
调用 Supabase signUp API
    ↓
检查返回结果
    ↓
如果成功：
  - 需要验证：提示用户查收邮件
  - 直接登录：跳转到个人档案
如果失败：
  - 显示错误信息
```
